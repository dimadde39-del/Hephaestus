# Phase 5.5C: Studio Advanced Views + Packaging + Polish

Phase 5.5C turns Studio into a coherent local app surface:

- Chat remains default and exact-message based.
- Workbench remains the practical agent-work inspection surface.
- Memory is now a first-class control surface for regular and strategic memory.
- Settings covers startup behavior, appearance, providers/models, trust/data,
  backup/export/restore, and advanced local details.
- Advanced views expose structured decision traces, Pareto frontiers, and QUBO
  formulations deliberately and secondarily.
- Onboarding is short and skippable.
- Packaging supports browser-served local Studio with static assets included in
  build artifacts.

- no Telegram;
- no voice;
- no cloud VM;
- no browser automation;
- no unrestricted terminal;
- no deploy or Git push;
- no Skill Forge, Capability Forge, RLM, or Context Forge;
- no fully autonomous coding loops.
